# 강의안(HTML) 재생성 프롬프트 — 대진대 바이브코딩 캠프

> **사용법:** 이 문서 전체를 다른 세션에 그대로 붙여넣으면, 맥락 없이도 강의 덱(단일 HTML)을 만들 수 있다.
> (선택) 기존 `docs/바이브코딩강의안.html`을 함께 첨부하면 비주얼 톤을 참고하게 할 수 있다.
> 이 문서는 강의안 ↔ 데모앱(`PRD.md` 등)을 잇는 단일 기준이기도 하다.

---

## 0. 너에게 주는 지시
너는 발표용 슬라이드 덱을 만드는 디자이너 겸 개발자다. 아래 **맥락·형식·구성·디자인**에 맞춰 **단일 HTML 파일** 강의 덱을 만들어라. 한국어. 비개발자(예비창업자)가 대상이다.

## 1. 강의 맥락
- 행사: **대진대학교 바이브코딩 창업캠프** (AI로 웹·앱 만들기, 1일)
- 일시: 2026-06-22(월) 10:00–18:00 / 강사 3명
- 대상: **코딩 경험 없는 예비창업자** 약 30명
- 핵심 메시지: "바이브코딩으로 비개발자도 **진짜 서비스**를 만들 수 있다 — 단, 도구보다 **기획**이 핵심이다."
- 스택(통일): **Cursor · Vercel · Supabase · 카카오 OAuth** — *기존 Google 스택(Gemini·Stitch·AI Studio·Cloud Run)은 쓰지 않는다.*

## 2. 산출물 형식
- **단일 `.html` 파일**, 외부 의존 최소(웹폰트 CDN만 허용).
- 슬라이드 = `<section class="slide">`. **← → / Space**로 이동, 우하단에 **현재/전체 번호**.
- 상단에 **챕터 라벨**(예: `CHAPTER 1 — 왜 지금인가?`).
- **16:9**, 프로젝터+노트북에서 읽히는 **큰 폰트**, 모바일에서도 깨지지 않게.
- 스크린샷 자리는 **플레이스홀더 박스**로: 점선 테두리 + 캡션(예: `[스샷: Supabase 테이블 화면]`). 실제 이미지는 나중에 끼움.
- 통계·수치 슬라이드엔 **출처 자리**를 남겨라(`출처: ___`). 추정치를 단정하지 말 것.

## 3. 디자인 시스템 (데모앱과 동일 브랜드)
- 로고: `logo_color.png` (대진대 공식 엠블럼, 블루+레드)
- 색: Primary `#0B4DA2` · Primary강조 `#083B7D` · Accent `#FF4D6D` · 배경 `#F7F8FA` · 표면 `#FFFFFF` · 본문 `#1A1D21` · 보조 `#6B7280` · 라인 `#E5E7EB`
- 폰트: **Pretendard** (CDN). 제목 700 / 본문 400–500
- 톤: 깔끔·또렷·자신감. 카드/여백 중심, 정보 위계 분명.

## 4. 슬라이드 구성 (이 순서대로)
> ⚠️ **마무리(섹션 8)는 실습(섹션 9) 뒤에 온다.** 실습으로 손에 결과물이 남은 상태에서 클로징.

### 오프닝
- **O-1 타이틀** — "2026 창업캠프 · 바이브코딩으로 앱 배포하기" / 대진대학교 / 일시 / 강사 3명
- **O-2 시작 전** — 카카오톡 오픈톡방 입장 + **캠프 앱 QR 입장**(대진대학교 AI캠퍼스)
- **O-3 강사 소개** — 3명

### CHAPTER 1 — 왜 지금인가? (기존 개발 vs 지금)
- **1-1 개발 패러다임의 전환** — 기계어→고급언어→노코드→AI. "더 많은 사람이 만들 수 있게 될 때마다 세상이 바뀌었다."
- **1-2 대 딸깍의 시대** — Stack Overflow 트래픽 쇠퇴 그래프. "검색해서 답 찾던 시대 → AI에게 또렷이 요청하면 만들어지는 시대." (출처 자리. *백준 이슈는 넣지 말 것*)

### CHAPTER 1 — 바이브코딩이란?
- **2-1 정의** — Andrej Karpathy(2025.2) 인용: "코드의 존재조차 잊고 감각에 맡기는 코딩."
- **2-2 BEFORE / AFTER** — 기존(기획→개발자 고용→3개월→수천만원) vs 바이브(하루·저비용).
- **2-3 지금 무슨 일이 벌어지나** — 개발자 수요 변화 + **leanaileaderboard.com**(AI로 운영되는 소규모 기업 리더보드) + 1인 창업가 수익 사례. (라이브 시연/캡처 자리)
- **2-4 실제 사례** — 비개발자 성공 사례(예: Storypot 등). 공통점 = "프롬프트를 잘 썼다."

### CHAPTER 1 — 진짜 다 "딸깍"되나? (반전)
- **3-1 전체 파이프라인** — 기획 · 고객 인터뷰 · 디자인 · 개발 · 퍼블리싱 · 마케팅. 그중 "딸깍"되는 건 **개발 일부뿐**.
- **3-2 그래서 창업자에게** — 아이디어 검증 비용·시간이 거의 0. 개발자 없이 MVP.
- **3-3 결론** — **핵심은 '기획'이다.**

### CHAPTER 1 — 알맞은 기획의 단계
- **4-1 음식 비유** — 개발=요리 / 기획=레시피 / 마케팅=먹게 하기 / 시장조사=니즈 파악. → 오늘은 **레시피(기획)**에 집중.
- **4-2 AI에게 잘 말하는 법** — "스터디 앱 만들어줘"(막연) vs 또렷한 요청 비교.
- **4-3 좋은 프롬프트 5요소** — 누가 쓰나 / 무엇을 / 왜 / 어떤 화면·기능 / 어떤 느낌.

### CHAPTER 2 — 데모 앱 소개 (기획→퍼블리싱 한 바퀴)
- **5-1 "이 강의용으로 만든 앱"** — 대진대학교 AI캠퍼스. **QR로 지금 입장**.
- **5-2 주요 기능** — ① 강의 자료 열람(로그인 X) ② 카카오 로그인 ③ 결과물 게시판(URL+스샷+한줄, 좋아요, 최신/인기순).
- **5-3 오늘의 흐름** — 기획 → 디자인 → 개발 → 배포 (도구: Cursor·Vercel·Supabase).

### CHAPTER 2 — 실제 개발 과정 (도그푸딩, 스샷 중심)
- **6-1 웹서비스 3층 구조** — 화면(Next.js) / 로직·인증 / 데이터(Supabase).
- **6-2 이게 PWA다** — 설치형 웹앱(홈 화면 추가).
- **6-3 이 앱을 이렇게 만들었다** — `[스샷: 기획 문서]` → `[스샷: Cursor 개발]` → `[스샷: Vercel 배포]` → `[스샷: Supabase 테이블/DB]`.

### CHAPTER 2 — 가이드라인 (툴 & 세팅)
- **7-1 AI 어시스턴트 비교** — 기획·에러 해결 파트너(ChatGPT/Claude/Gemini 등).
- **7-2 오늘 쓸 도구 4종 역할** — 기획(챗) / 개발 **Cursor** / 배포 **Vercel** / DB·인증 **Supabase**.
- **7-3 세팅 개요(오후 예고)** — 설치·가입·연결을 한 줄씩.

### ☕ 중식 (구분 슬라이드)

### CHAPTER 3 — 식사 후 실습 (실전 핸즈온)
- **9-1 실습 시작** — 각 단계는 강사가 먼저 시연 → 따라하기. 보조강사 2명이 지원.
- **9-2 환경 준비** — Cursor 설치 + Node + git 초기화.
- **9-3 STEP 0 — 선배포** — 빈 앱이라도 먼저 **Vercel** 배포 성공시키기.
- **9-4 STEP 1 — 기획** — 챗으로 PRD 정리 + **프롬프트 치트시트**(복붙용).
- **9-5 STEP 2 — 디자인** — 레퍼런스(Mobbin) → **Cursor에 "이 느낌으로"** 지시.
- **9-6 STEP 3 — 개발** — Cursor로 화면·기능 구현.
- **9-7 STEP 4 — DB 연결** — **Supabase** 테이블 + **카카오 로그인**.
- **9-8 QA · 개선** — 직접 눌러보며 점검, 한 번에 하나씩 보완.
- **9-9 STEP 5 — 배포** — **Vercel** 재배포.
- **9-10 도메인 연결** — 시연 + 가이드(전파 대기 안내).
- **9-11 결제** — "이런 게 있다"만(토스페이먼츠/포트원). *앱엔 미구현.*
- **9-12 트러블슈팅** — 막히면 이 순서로 → 안 되면 오픈톡방에 캡처.
- **9-13 홈 화면에 추가** — 진짜 앱처럼 완성.
- **9-14 결과물 게시판에 업로드** — 내가 만든 앱을 캠프 앱에 올리기(앱으로 회귀).
- **9-15 (참고) 앱스토어 등록** — PWA도 스토어 등록 가능, 오늘은 과정만.

### CHAPTER 3 — 마무리 (클로징, 실습 뒤)
- **8-1 메이플 전직 비유** — AI 활용의 진화: 일상 질문 → 콘텐츠 생성 → 코드 어시스턴트 → **바이브코딩** → 에이전틱 워크플로우 → 오케스트레이션. (오늘은 '바이브코딩' 단계)
- **8-2 반복이 다음 단계를 연다** — 오늘 과정을 기억하고 반복 → 자동화 워크플로우 → 오케스트레이션. "오늘은 1차 전직."
- **8-3 다 같이 결과물 구경 + 마무리 메시지** — 게시판에 모인 결과물을 함께 보고, "**도구는 바뀌어도, 무엇을 만들지 또렷하게 정리하는 능력은 남는다.**"
