# 유저플로우 — [내 앱 이름]

> 사용자가 어떤 화면을 어떤 순서로 이동하는지 정리합니다.

## 1. 주요 사용자 흐름

```text
첫 방문
  -> 홈
  -> [핵심 화면]
  -> [상세/작성/신청 등]
  -> 완료
```

## 2. 화면 목록

| 화면 | 하는 일 | 로그인 필요? | 다음 이동 |
| --- | --- | --- | --- |
| 홈 | 앱 소개와 핵심 행동 안내 | 아니오 | [목록/작성/로그인] |
| [목록 화면] | [데이터를 카드로 보여줌] | [예/아니오] | [상세] |
| [상세 화면] | [자세한 내용 표시] | [예/아니오] | [수정/삭제/뒤로] |
| [작성 화면] | [새 항목 등록] | 예 | [완료 후 목록] |
| 로그인/회원가입 | 사용자 인증 | - | [이전 화면] |

## 3. 핵심 시나리오

### 시나리오 A. 처음 온 사용자가 핵심 행동을 완료한다

1. 홈에 들어온다.
2. 앱의 목적을 이해한다.
3. [핵심 버튼]을 누른다.
4. 필요한 정보를 입력한다.
5. 완료 화면 또는 목록으로 이동한다.

### 시나리오 B. 로그인 사용자가 본인 데이터를 관리한다

1. 로그인한다.
2. 본인이 작성한 항목을 확인한다.
3. 수정하거나 삭제한다.

## 4. 예외 흐름

- 데이터가 없을 때: 빈 상태 문구와 다음 행동 버튼을 보여준다.
- 로그인이 필요할 때: 로그인 화면으로 안내한다.
- 저장 실패 시: 사용자가 이해할 수 있는 오류 메시지를 보여준다.
