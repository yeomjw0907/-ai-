# 디자인 시스템 — [내 앱 이름]

> AI가 화면을 만들 때 흔들리지 않도록 색, 글자, 버튼, 카드 규칙을 정리합니다.

## 1. 디자인 키워드

- 분위기: [예: 깔끔한 / 귀여운 / 전문적인 / 활기찬]
- 참고 서비스: [예: 토스, 당근, 에브리타임]
- 피해야 할 느낌: [예: 너무 복잡한 화면]

## 2. 색상

| 역할 | 색상 | 용도 |
| --- | --- | --- |
| Primary | [예: #0B4DA2] | 주요 버튼, 강조 링크 |
| Accent | [예: #FF4D6D] | 좋아요, 경고, 포인트 |
| Background | [예: #F7F8FA] | 전체 배경 |
| Surface | [예: #FFFFFF] | 카드, 입력 영역 |
| Text | [예: #111827] | 주요 글자 |

## 3. 폰트와 글자 크기

- 기본 폰트: [예: Pretendard, system-ui]
- 제목: 굵고 짧게
- 본문: 14~16px, 줄간격 넉넉하게
- 버튼 글자: 짧고 행동 중심으로 작성

## 4. 컴포넌트 규칙

| 컴포넌트 | 규칙 |
| --- | --- |
| 버튼 | 높이 48px 이상, 둥근 모서리, 한 화면에 주요 버튼은 1개 |
| 카드 | 흰 배경, 얕은 그림자, 12~16px radius |
| 입력창 | 라벨 또는 placeholder를 명확하게 표시 |
| 하단 탭 | 주요 화면 3~5개만 배치 |
| 빈 상태 | 사용자가 다음에 할 행동을 안내 |

## 5. 모바일 기준

- 최대 폭: 480px
- 터치 영역: 최소 44~48px
- 긴 설명보다 카드와 리스트 중심
- 중요한 행동은 화면 하단 또는 상단 오른쪽에 배치
