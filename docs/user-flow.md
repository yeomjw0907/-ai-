## 2. 주요 플로우

### 플로우 A — 자료만 보기 (비로그인)

QR/링크 진입 → **S1 홈** → 탭으로 흐름·치트시트·링크 열람 → (원하면) **S2 게시판** 구경

### 플로우 B — 결과물 올리기 (핵심)

**S2 게시판** → `글쓰기` → 비로그인이면 **S5 로그인** (계정 없으면 **S6 회원가입**) →
**S4 글쓰기**(URL·스샷·제목·한줄) → 게시 → **S3 상세**로 이동

### 플로우 C — 구경 & 좋아요

**S2 목록** (최신순/인기순 토글) → 카드 클릭 → **S3 상세** → ♡ 좋아요 (로그인 필요 시 S5 유도)

### 플로우 D — 운영

작성자: **S3** → 수정(S4) / 삭제
강사(admin): 모든 글 **S3**에서 삭제 가능

## 3. 회원가입 / 로그인 상세

- **회원가입(S6)**: 학번·이름·과·비밀번호·비밀번호 확인 입력
  - 클라이언트에서 **비밀번호 == 비밀번호 확인** 검사 후 제출
  - 내부적으로 `이메일 = 학번@ai-campus.local`로 Supabase 가입 → 성공 시 `profiles`에 학번·이름·과 저장
  - 같은 학번이면 "이미 가입된 학번입니다" 안내(Supabase 중복 에러)
- **로그인(S5)**: 학번·비밀번호 → `이메일 = 학번@ai-campus.local`로 변환해 로그인
- 별도 온보딩 화면 없음 (이름·과를 가입 시 함께 받음)

## 4. 네비게이션

- 하단 탭(모바일): **자료(S1)** · **게시판(S2)** · **내 정보/로그인**
- 게시판 우하단: 글쓰기 FAB(+)

## 5. 빈/예외 상태

- 게시판 글 0개 → "첫 결과물을 올려보세요" 안내
- 비로그인 글쓰기 시도 → 로그인 유도(모달 또는 S5 이동)
- 비밀번호 불일치 → 가입 버튼 비활성/경고
- 학번 중복 → "이미 가입된 학번입니다"
- 로그인 실패(학번/비번 불일치) → "학번 또는 비밀번호를 확인하세요"
- 이미지 업로드 실패 → 재시도 안내
