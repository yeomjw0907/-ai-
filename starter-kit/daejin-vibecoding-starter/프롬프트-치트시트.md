# 📋 프롬프트 치트시트 (복붙용)

> Cursor 채팅이나 ChatGPT/Claude에 **그대로 붙여넣고**, 대괄호 [ ]만 내 내용으로 바꾸세요.
> 핵심: **한 번에 하나씩**, 또렷하게 요청하기.

---

## 1단계 · 기획 (레시피 쓰기)
```
나는 비개발자야. 아래 아이디어로 앱을 만들 거야.
- 누가 쓰나: [대상 사용자]
- 무엇을: [핵심 기능]
- 왜: [해결할 문제]
- 어떤 화면: [예: 홈/목록/상세/글쓰기]
- 어떤 느낌: [예: 깔끔하고 신뢰감 있는]

이걸 PRD(기획 문서)로 정리해줘. 핵심 기능을 F1, F2, F3로 나누고
각 화면과 화면 이동 흐름도 표로 만들어줘.
```

## 2단계 · 디자인
```
[참고 이미지나 사이트 주소]를 첨부할게. 이 느낌으로 만들고 싶어.
색은 [메인색]/[강조색], 모바일 우선, 카드 중심으로.
먼저 홈 화면부터 Tailwind로 만들어줘. (한 번에 한 화면만)
```

## 3단계 · 개발 (하나씩)
```
이제 [기능 이름]을 만들어줘. 예: "글 목록을 카드로 보여주는 화면"
- 동작: [어떻게 동작해야 하는지]
- 완성되면 어떻게 확인하는지도 알려줘.
다른 기능은 아직 건드리지 마.
```

## 4단계 · DB 연결 (Supabase)
```
Supabase를 연결할 거야. [저장할 데이터: 예 - 글(제목,내용,작성자), 좋아요]
1) 필요한 테이블 SQL을 만들어줘.
2) 앱에서 이 데이터를 읽고 쓰는 코드를 붙여줘.
3) 키는 .env.local 에서 불러오게 해줘. 코드에 직접 넣지 마.
```

## 5단계 · 배포 (Vercel)
```
이 프로젝트를 GitHub에 올리고 Vercel로 배포하는 순서를
비개발자도 따라할 수 있게 한 단계씩 알려줘.
중간에 입력해야 하는 값이 뭔지도 짚어줘.
```
