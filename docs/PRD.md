# PRD — 대진대학교 AI캠퍼스

> 바이브코딩 창업캠프 데모 앱 · 기획 문서(Product Requirements Document)

## 1. 개요
| 항목 | 내용 |
|---|---|
| 제품명 | **대진대학교 AI캠퍼스** |
| 한 줄 정의 | 바이브코딩 창업캠프 참가자를 위한 **강의 자료 + 결과물 공유 게시판** PWA |
| 플랫폼 | PWA (모바일 우선) · 홈 화면 설치형 |
| 기술 스택 | Next.js → Vercel 배포 · Supabase(DB·Auth·Storage) · 카카오 OAuth |
| 타깃 | 대진대 캠프 참가자(비개발자) 약 30명 내외 |
| 작성일 | 2026-06 |

## 2. 목적 (왜 만드나)
1. **강의 동반** — 참가자가 QR로 입장해 오늘의 흐름·치트시트·도구 링크를 한 곳에서 본다.
2. **결과물 공유** — 각자 만든 앱을 올리고 서로 구경 → 마무리에 "다 같이 보기" 클라이맥스.
3. **강의의 도그푸딩 데모** — 이 앱 자체가 강의 6·7번(실제 개발 과정 / 가이드)의 살아있는 예시.

## 3. 핵심 기능

### F1. 강의 자료 (로그인 불필요)
- 오늘의 흐름 · 타임테이블
- 프롬프트 치트시트 (복붙용)
- 도구 링크 모음 (Cursor · Vercel · Supabase · 카카오 등)

### F2. 인증
- **카카오 로그인** (Supabase Auth · Kakao provider)
- **최초 로그인 시 표시 이름 입력**(온보딩) → 이후 게시글에 이 이름이 표시된다.
- 로그아웃

### F3. 결과물 게시판
| 세부 | 내용 |
|---|---|
| 글 작성 항목 | 앱 **URL** + **스크린샷 이미지** + **제목 / 한줄설명** |
| 열람 권한 | **누구나**(비로그인 포함) |
| 작성 권한 | **카카오 로그인 사용자** |
| 상호작용 | **좋아요(♡)** — 사용자당 글 1회 |
| 목록 정렬 | **최신순 / 인기순(좋아요수) 토글** |
| 작성자 표시 | 가입 시 입력한 표시 이름 |
| 운영 권한 | 작성자 **수정·삭제** / 강사(admin) **삭제** |

## 4. 화면 흐름 (요약 — 상세는 `user-flow.md`)
홈/자료(비로그인) → 게시판 목록(카드 갤러리·정렬 토글) → 글 상세(스샷·링크·좋아요) → 글쓰기/수정(로그인) → 로그인 + 표시이름 온보딩

## 5. 범위 (Scope)
- **In:** F1 ~ F3
- **Out (오늘 안 함):** 결제, 댓글, 알림, 팔로우, 검색, 다국어
  - *결제는 강의에서 "이런 게 있다"만 언급, 앱엔 미구현.*

## 6. 성공 기준
- 캠프 중 참가자 **다수가 결과물 1건 이상 업로드**
- 시연 중 로그인 → 글쓰기 → 게시 흐름이 **막힘 없이 동작**
- 마무리에 게시판이 결과물로 **꽉 찬 화면**이 나온다.

## 7. 데이터 (요약 — 상세는 `database-schema.md`)
`profiles`(사용자·표시이름·admin) · `posts`(url·이미지·제목·설명·작성자) · `likes`(좋아요) · Storage `post-images` 버킷
